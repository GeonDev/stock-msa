# 🤖 Gemini CLI Project Instructions

이 파일은 Gemini CLI가 이 프로젝트에서 작업을 수행할 때 반드시 지켜야 할 **최우선 지침**을 담고 있습니다.

## 🚨 보안 및 환경 변수 관리 (절대 금지 사항)
- **환경 변수 파일 보호:** 실제 API 키나 비밀번호가 포함된 `.env`, `env` (점이 없는 파일 포함) 및 모든 `.env.local` 파일은 절대로 깃에 스테이징(`git add`)하거나 커밋하지 마세요.
- **템플릿 파일 허용:** 실제 값이 없는 템플릿 파일(`.env.example`, `config.template` 등)은 프로젝트 설정을 위해 커밋이 허용됩니다.
- **실제 키 노출 차단:** 템플릿 파일이라 하더라도 실수로 실제 키를 기입하여 푸시하는 일이 없도록 커밋 전 내용을 반드시 확인하세요.
- **파일 이름 변칙 금지:** 보안 검사를 피하기 위해 실제 정보가 담긴 `.env` 파일의 이름을 바꾸어 푸시하는 행위는 엄격히 금지됩니다.
