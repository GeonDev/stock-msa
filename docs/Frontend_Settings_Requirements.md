# 프론트엔드 설정(Settings) 요구사항 정의서

## 1. 개요
본 문서는 Phase 4(프론트엔드 대시보드) 구축 과정에서 구현될 '설정(Settings)' 페이지의 기능적 요구사항을 정의합니다. 설정 페이지는 시스템 운영(배치 제어), 퀀트 환경 설정, 그리고 인프라 모니터링의 통합 제어 센터 역할을 합니다.

---

## 2. 주요 설정 카테고리

### 2.1 UI/UX 테마 설정 (Visual Experience) - [MUST HAVE / Priority 1]
사용자 환경에 따른 시각적 최적화를 제공합니다.
- **다크/라이트 모드 토글**: 
  - **다크 모드**: bg-zinc-950 기반, 고대비 네온 그린/레드 포인트 (기본값).
  - **라이트 모드**: 깨끗한 화이트 배경, 눈의 피로를 줄인 소프트한 색상 테마 적용.
  - **시스템 동기화**: OS 설정에 따라 자동으로 모드 전환.
- **테마 영속성**: 브라우저를 닫아도 설정이 유지되도록 LocalStorage 연동.

### 2.2 시스템 및 인프라 모니터링 (Infrastructure)
MSA 아키텍처의 특성상 각 서비스의 생존 여부와 연결 상태를 시각화합니다.
- **서비스 헬스 체크**: `stock-corp`, `stock-price`, `stock-finance`, `stock-strategy` 등 마이크로서비스의 실시간 상태(UP/DOWN) 및 버전 정보 표시.
- **Eureka 서버 상태**: 서비스 디스커버리에 등록된 인스턴스 목록 및 메타데이터 조회.

### 2.3 데이터 관리 및 배치 제어 (Data & Batch Control)
백엔드 API를 수동으로 트리거하여 데이터를 강제 동기화하는 기능을 제공합니다.
- **배치 수동 실행 (Manual Trigger)**:
  - 기업 정보/업종 수집 (`Step 1, 2`)
  - 주가 데이터 수집 (KOSPI/KOSDAQ/KONEX별 선택)
  - 재무 데이터 수집 (분기별 선택 실행)
- **DART API 쿼터 모니터링**: 일일 10,000건 한도 중 현재 사용량 및 잔여량 표시.

### 2.4 퀀트 전략 기본값 설정 (Strategy Defaults)
- **거래 비용 설정**: 기본 매매 수수료율, 증권거래세율, 슬리피지 모델.
- **자산 배분 제약**: 단일 종목 최대 비중 제한, 최소 거래 단위 설정.

## 4. 기술적 고려사항
- **보안**: 설정 변경 권한은 추후 Phase 6의 관리자 권한(RBAC)과 연계되어야 함.
- **비동기 처리**: 배치 실행 트리거 시 브라우저 타임아웃을 방지하기 위해 Polling 또는 WebSocket으로 진행률 표시 권장.
- **영속성**: 사용자 설정값은 프론트엔드 LocalStorage에 저장하거나, 서버 사이드에 `UserPreference` 테이블을 신설하여 관리.
