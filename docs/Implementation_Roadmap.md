## Phase 1: 데이터 품질 검증 ✅ (완료)

**목표**: 수집된 데이터의 정합성 및 완전성 확보

### 1.1 데이터 정합성 검증 ✅ (완료)
- [x] 필수 필드 누락 검사 (TotalAsset, Revenue 등 `FinanceValidationService` 구현)
- [x] 대차대조표 등식 검증 (자산 = 부채 + 자본, 오차율 0.1% 이내)
- [x] 이상치 탐지 (전분기 대비 자산/매출 변동폭 1000% 이상 탐지 및 `WARN_OUTLIER` 부여)
- [x] 검증 상태 관리 (`VERIFIED`, `ERROR_MISSING`, `ERROR_IDENTITY` 등 상태 세분화)

### 1.2 성능 모니터링 ✅ (완료)
- [x] DART API Rate Limiting 구현 (Sliding Window 방식, 분당 1,000회 제한 준수)
- [x] 배치 처리 단계별 상세 시간 측정 및 로깅
- [x] 배치 인서트 최적화: `rewriteBatchedStatements=true` 및 `jdbc.batch_size` 적용 (성능 10배 향상)

### 1.3 데이터 보완 및 최적화 ✅ (완료)
- [x] DART API 일일 한도 대응 (분기별 개별 수집 API 구현으로 호출 수 75% 절감)
- [x] DART Corp Code DB 기반 관리 및 상장사 매핑 자동화 (`SectorUpdateBatch`)
- [x] 주가 데이터 누락일 탐지 및 자동 재수집 로직 (기존 1일에서 최근 7일치 배치 수집으로 확대)
- [x] 기술적 지표 계산 성공률 향상 (히스토리 부족 종목 처리 개선)

## Phase 5: AI 인사이트 및 리서치 ✅ (완료)

**목표**: 퀀트 데이터와 LLM을 결합한 지능형 투자 보조 및 자동 리서치 시스템

### 5.1 AI 리서치 보고서 자동 생성 (LLM 연동) ✅
- [x] **자동 재무 분석**: DB 내 재무제표 데이터를 LLM 프롬프트로 변환하여 자연어 분석 리포트 생성
- [x] **DART 공시 RAG**: DART 공시 원문을 수집/벡터화(pgvector)하여 핵심 리스크 및 전망 요약 응답
- [x] **시각화 리포트**: `JFreeChart` 기반 재무 지표(PER/PBR/ROE 등) 바 차트 이미지 자동 생성
- [x] **멀티모달 응답**: AI 분석 텍스트와 차트 이미지를 결합한 통합 메시지 전송 시스템 구축

### 5.2 텔레그램 리포팅 및 지능형 알림 ✅
- [x] **텔레그램 봇 연동**: Spring Boot 기반 Telegram Bot API 통합 및 대화형 명령 처리기 구현
- [x] **개인화 스케줄러**: 사용자별 관심종목(Watchlist) 기반 08:30 아침 브리핑 및 16:00 장 마감 리포트 자동화
- [x] **온디맨드 리서치**: `/report [종목코드]` 명령어 입력 시 즉시 AI 분석 보고서 및 차트 응답
- [x] **지능형 알림**: 특정 지표 조건(예: PER 10 이하) 도달 시 실시간 텔레그램 푸시 알림 (`/alert`)
- [x] **웹 관리 UI**: 프론트엔드 `Settings` 화면에서 관심 종목 및 활성 알림 실시간 모니터링/삭제 연동

## Phase 6: 머신러닝 전략 (계획) ⏳

**목표**: 머신러닝 기반 종목 예측 및 포트폴리오 최적화

### 6.1 퀀트 피처 엔지니어링 및 예측 모델 ⏳
- [ ] **피처 엔진**: 기술적(Price/Volume) + 재무(Fundamental) 지표를 결합한 통합 시계열 데이터셋 구축
- [ ] **수익률 예측**: XGBoost/LightGBM 기반 상위 수익률 종목 예측 및 분류 모델 개발
- [ ] **포트폴리오 최적화**: 강화학습(DQN) 기반 자산별 동적 비중 조절 로직 고도화

### 6.2 투자 감성 분석 및 뉴스 데이터 ⏳
- [ ] **뉴스 센티먼트**: 외부 뉴스 API 연동을 통한 종목별 긍/부정 감성 지수 산출
- [ ] **트렌드 감지**: 섹터별 뉴스 빈도 분석을 통한 시장 주도 테마 자동 감지 및 알림
