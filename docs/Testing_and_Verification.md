## 발견된 이슈 및 조치 사항

### 🛠 시스템 통합 및 통신 (2026-03-04 수정)
- **URL 프리픽스 불일치**: `stock-strategy`와 `stock-finance`가 `stock-price`를 호출할 때 Gateway용 프리픽스(`/api/v1/...`)를 포함하던 버그 수정. 서비스 간 직접 통신은 `/internal/...` 경로 사용.
- **날짜 규격 불일치**: `yyyyMMdd`와 `yyyy-MM-dd` 혼용으로 인한 파싱 에러 해결. 모든 서비스에서 `DateUtils.toStringLocalDate`를 사용하여 유연하게 처리.
- **배치 요청 청킹**: 수천 개의 종목 코드를 URL 파라미터로 보낼 때 발생하는 `414 Request-URI Too Long` 및 `Fatal Error` 방지를 위해 200개 단위로 Chunking 처리.

### 🛠 데이터 직렬화 및 캐싱 (2026-03-04 수정)
- **Redis LocalDate 직렬화**: `Jackson2JsonRedisSerializer`에 `JavaTimeModule`이 등록되지 않아 `LocalDate` 필드에서 터지던 문제 해결 (`RedisConfig` 수정).
- **Redis Null 캐싱**: `disableCachingNullValues()` 설정으로 인해 데이터가 없을 때(`null`) 에러가 발생하던 문제 해결.

### 🛠 백테스팅 엔진 (2026-03-04 수정)
- **리밸런싱 날짜 버그**: 매월 1일이 휴일일 경우 한 달 전체 리밸런싱을 건너뛰던 로직 수정. 시작일(`startDate`)은 무조건 리밸런싱을 수행하도록 보완.
- **자가 주입(Self-Injection)**: 동일 클래스 내 `@Async` 호출 시 프록시가 작동하지 않아 동기/트랜잭션 문제가 생기던 구조를 자가 주입(`self.runSimulationAsync`)으로 개선.
- **중복 데이터 처리**: `FactorScoringService`에서 재계산 시 기존 데이터를 삭제(`deleteByScoreDate`)하고 `flush()`하여 중복 키 에러 방지.
- **NPE 방지**: `PortfolioHolding`의 수량 및 금액 필드를 0으로 초기화하여 첫 매수 시 발생하던 NullPointerException 해결.
